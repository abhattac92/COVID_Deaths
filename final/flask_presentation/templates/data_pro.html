We started with 700 different csv files of webscraped data from the kickstarter webpage. These files were combined to create one large csv. This file was stored in AWS and linked to a postgres server in PGadmin. We are able to access this large file in the cloud saving local data. We initially had 2,359,293 rows and 41 columns. The dataset contains projects added to Kickstarter between 22 April 2009 and 16 September 2020. We dropped columns with small no-null values and irrelevant columns listed below as well

converted_pledged_amount - same as usd_pledged
country - same as country_displayed_name
creator - irrelevant
currency - irrelvant as converted
currency_symbol - irrelevant
currency_trailing_code - irrelevant
currenct_currency - irrelevant as it has converted
disable_communication - almost all the same
fx_rate - irrelevant. info in USD_pledged
is_starrable - irrelevant
name - irrelevant
photo - irrelevant
profile -irrelevant
slug -same as name. irrelevant
source_url- irrelevant
spotlight - irrelevant as it is only for projects that succeeded. will skew results
state_changed_at - irrelevant
static_usd_rate -irrelevant
urls - irrelevant
usd_type - irrelevant

The next step was to convert the Unix time to standard time. We also converted the blurb into a numerical number to see if it is relevant in predicting. We converted the words to a length number. We also replaced all null values with zero and none depending on the column. We then broke down the converted time and created columns for the month and dates. We also created two hour time increments for data processing purposes. We also took out any data whose state was cancelled, suspended, or still live as this would not give us any insights.
